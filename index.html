How JavaScript works?

As we know JavaScript is an interpreted programming language which engines are inbuilt in all the modern browsers today. When we try to load the js file in the browser, the JavaScript engine will execute each line of the file from top to bottom. It is a single-threaded programming language, which means it has a single Call Stack and it can do one thing at a time. basically, JavaScript runs inside an environment called JavaScript Runtime Environment along with many other components. Here JavaScript Engine consists of two main components like Memory Heap ( this is where the memory allocation happens) and Call Stack (this is where your stack frames are as your code executes). If we want to execute heavy code without blocking the UI we have a solution like asynchronous callbacks in JavaScript.

Difference Between Local Storage and Session Storage

In The localStorage and sessionStorage, there is one major difference between is the expiration date. basically, localStorage persists the storage across browser sessions. So even if the browser is opened in a new tab, or the browser is closed and reopened, the storage is saved, there is no expiration. on the other side, sessionStorage is cleared when the page session ends. For the local storage and session storage, we can set, get, remove, and clear items using the appropriate methods like setItem(),getItem(),removeItem(),clear() etc.
